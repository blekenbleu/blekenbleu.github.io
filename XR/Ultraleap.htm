First Use
Ultraleap service, installed by VR1 tool, launches at Windows boot
  • did not detect when VR1 was powered on
  • restarting Ultraleap service did not resolve
  • Reset Headset in VR1 tool resolved it:

      system tray icon

  • Launch Ultraleap Control Panel:


  • Ultraleap Control Panel display works with or without SteamVR:


  • Launching VR1 tool with VR1's Ultraleap Control Panel display open
    makes Mixed Reality unhappy:

    - but mixed reality headset button still works;
      starting Ultraleap Control Panel after VR1 tool is ok
  • Installing Steam ultraleap hyperion initially disabled VR1 Mixed Reality.
    That Steam installation replaced VR1's Ultraleap runtime.
    - Steam ultraleap hyperion displayed Ultraleap Control Panel in headset.
    - Neither Ultraleap Hyperion's Microgestures nor Launcher app ran.
  • Reinstalling VR1 tool's Ultraleap Runtime restored Mixed Reality function
  • Restoring compatible Ultraleap Control Panel function
    required uninstalling SteamVR ultraleap hyperion
  • Ultraleap's Aurora game eventually loads and plays
    with SteamVR's Ultraleap Hyperion runtime,
    but if representative of Ultraleap hand tracking,
    then it for me was a hateful waste of money and effort.
  • Since hand tracking motion controller emulation is IMO much worse,
    seemingly touchscreen emulation with headset gaze pointing is wanted.
