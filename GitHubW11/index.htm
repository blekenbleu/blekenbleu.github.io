GitHub Desktop and Jekyll on Windows 11

Differences in Jekyll experiences on different Windows versions is mostly about Ruby and Jekyll evolution.
Ruby installation has become more trouble-free, without hangs and need for retries.
Jekyll, on the other hand, seems IMO to have regressed:
  • <details> seemingly are no longer supported in markdown
  • markdown defaults to blog-specific format
  • ignores markdown files lacking front matter starting on the first line

    This last issue is a distinct difference between Windows Jekyll and whatever GitHub uses to process markdown.
    Fortunately, empty front matter suffices, and a ### heading immediately following overrides the default title.
    My pages include over 60 markdown files that needed editting for this issue;
  • starting over, I would completely avoid markdown for GitHub Pages;
  • trivial changes typically require over 5 minutes for GitHub to process.

    Ruby updates

    Gems can be updated, but Ruby itself as installed by RubyInstaller cannot be updated, only replaced.
    While Ruby is here used only for Jekyll, its bundled MSYS2 installation gets used a lot, with numerous packages added.
    Before replacing Ruby, run MSYS2 package manager to document its current package list:
  • `pacman -Q > /d/oldMSYS2.txt`
    .. then compare with a list from new MSYS2 to determine what packages had been added.
    Of course, any MSYS2 home/user folder customizations should also be ported.
